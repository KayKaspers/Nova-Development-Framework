# Nova Development Framework

> **Plan. Build. Review. Improve.**

The **Nova Development Framework (NDF)** is a modular software engineering framework for planning, building, reviewing, documenting and continuously improving modern software projects with human-led, AI-assisted workflows.

NDF is designed for projects that should not grow chaotically, but follow clear architecture, documentation, quality and release standards from the beginning.

---

## What is NDF?

NDF is not only a documentation repository.

It is a practical framework for:

- project planning
- architecture decisions
- documentation-first development
- AI-assisted implementation
- prompt standardization
- project quality gates
- release readiness
- project knowledge management
- reusable templates
- beginner-friendly software workflows

---

## Core Principle

> **AI creates. Humans approve.**

AI assistants such as Claude, ChatGPT or other tools may help create code, documentation and analysis.  
Repository history, releases and critical decisions remain under human control.

---

## The Six NDF Pillars

```text
NDF
├── Foundation
├── Academy
├── Toolkit
├── AI Collaboration
├── Delivery
└── Best Practices
```

### Foundation

Rules, governance, standards, ADRs, Project Brain, quality gates and release policies.

### Academy

Beginner-friendly learning material, tutorials, exercises and long-form documentation.

### Toolkit

Templates, prompt libraries, checklists, scripts and future generators.

### AI Collaboration

Structured collaboration between humans and AI assistants.

### Delivery

The complete path from idea to release.

### Best Practices

Lessons learned from real projects such as CastCore, SpeakCore, AirCore and SC-OrgaSuite.

---

## Recommended Beginner Workflow

```text
Nova / ChatGPT
  plans, reviews and creates prompts

Claude
  implements small work packages

VS Code
  is used as editor

GitHub Desktop
  is used for commit, pull and push

GitHub
  stores the project history
```

Cursor and advanced API-based workflows are optional, not required.

## Repository Structure

```text
academy/          Learning material and beginner guides
adr/              Architecture Decision Records
branding/         Brand identity and design standards
docs/             Framework documentation
framework/        Standards, templates, prompts and models
project-brain/    Long-term project knowledge
scripts/          Utility scripts
standards/        General standards
templates/        Reusable templates
```

This structure will continue to evolve toward the long-term NDF module architecture.
