# NDF Standard – Health Score Update Rules

## Regeln

1. Score nicht im selben Commit wie riskanten Code-Fix ändern.
2. Score erst nach Evidenz/Test/CI anpassen.
3. Deltas begründen.
4. Offene Risiken sichtbar lassen.
5. Review-only erhöht Security nicht automatisch.
6. Security steigt nur nach Mitigation oder verifizierter Control.
7. Keine Score-Inflation.

## Pflichtinhalt

- Work Package ID
- alter Wert
- neuer Wert
- Deltas
- Begründung
- Evidenz
- offene Risiken
- nächste Work Packages
