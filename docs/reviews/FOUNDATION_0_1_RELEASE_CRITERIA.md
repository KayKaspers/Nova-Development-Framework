# Foundation 0.1 Release Criteria

## Pflichtkriterien

Foundation 0.1 darf erst als abgeschlossen gelten, wenn:

- [ ] README erklärt das Projekt verständlich.
- [ ] Constitution vorhanden.
- [ ] Governance vorhanden.
- [ ] Blueprint vorhanden.
- [ ] Domain Model vorhanden.
- [ ] Practical Workflow vorhanden.
- [ ] Prompt Library v0.1 vorhanden.
- [ ] Template Library v0.1 vorhanden.
- [ ] Project System v0.1 vorhanden.
- [ ] Toolkit v0.1 vorhanden.
- [ ] Project Starter Manual Flow vorhanden.
- [ ] Academy Band 1 Skeleton vorhanden.
- [ ] Export Concept vorhanden.
- [ ] Website Concept vorhanden.
- [ ] ADRs für Hauptentscheidungen vorhanden.
- [ ] Project Brain aktualisiert.
- [ ] Repository-Readiness-Dateien vorhanden.
- [ ] Branding Bible v0.1 vorhanden.
- [ ] Minimal Example Project vorhanden.
- [ ] Foundation 0.1 Release Notes finalisiert.

## Optionale Kriterien

- [ ] Basic Markdown-Lint-Konzept
- [ ] Link-Check-Konzept
- [ ] MkDocs lokaler Build-Test
- [ ] erstes DOCX/PDF Export-Experiment

## Release-Entscheidung

Foundation 0.1 ist ein **Foundation Release**, kein vollständiger Produktrelease.

Ziel ist Stabilität des Fundaments, nicht Vollständigkeit aller Inhalte.
