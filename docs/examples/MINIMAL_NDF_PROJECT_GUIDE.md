# Minimal NDF Project Guide

## Zweck

Dieses Dokument erklärt, warum das Minimal NDF Project existiert und wie es genutzt werden kann.

## Warum ein Beispielprojekt?

Frameworks werden verständlicher, wenn man ein vollständiges Beispiel sieht.

Das Minimal NDF Project zeigt:

- wie ein Project Manifest aussieht
- wie Project Brain genutzt wird
- wie Work Packages aussehen
- wie Claude-Prompts formuliert werden
- wie Rückmeldung an Nova funktioniert
- wie Quality Gates klein starten können

## Wie nutzt man das Beispiel?

1. Öffne `examples/minimal-ndf-project/README.md`.
2. Lies Project Manifest und Project Profile.
3. Prüfe die Work Package Queue.
4. Öffne den ersten Claude-Prompt.
5. Vergleiche danach die Rückmeldung an Nova.

## Für welche Projekte ist das relevant?

Für jedes neue Projekt im NDF-Ökosystem:

- Referenzprojekt B (CI/Docker)
- Referenzprojekt A (Destructive Operations)
- zukünftige Referenzprojekte
- kleinere Tools
- Dokumentationsprojekte

## Wichtig

Das Beispiel ist kein Zielzustand für große Projekte. Es ist ein Startpunkt.
