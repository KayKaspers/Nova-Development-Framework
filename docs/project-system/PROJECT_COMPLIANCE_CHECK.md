# Project Compliance Check

## Zweck

Der Compliance Check prüft, ob ein Projekt NDF-Regeln erfüllt.

## Bereiche

- README
- Project Brain
- Roadmap
- ADRs
- Standards
- Prompt Workflow
- Quality Gates
- Security
- Tests
- Release Notes

## Ergebniswerte

- pass
- warning
- fail
- not_applicable

## Beispiel

| Bereich | Status | Empfehlung |
|---|---|---|
| README | pass | keine |
| Project Brain | warning | aktualisieren |
| ADRs | fail | erste ADRs erstellen |
| Tests | warning | Teststrategie dokumentieren |

## Ausgabe

Ein Compliance Check erzeugt:

- Score
- Risiken
- empfohlene Work Packages
