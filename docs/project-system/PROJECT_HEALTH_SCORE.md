# Project Health Score

## Zweck

Der Health Score bewertet den Zustand eines Projekts.

## Dimensionen

| Dimension | Gewicht |
|---|---:|
| Architecture | 15 |
| Documentation | 15 |
| Testing | 15 |
| CI/CD | 10 |
| Security | 10 |
| Deployment | 10 |
| Project Brain | 10 |
| Prompt Workflow | 5 |
| Release Readiness | 10 |

## Bewertung

Jede Dimension erhält 0 bis 100 Punkte.

## Interpretation

- 90–100: exzellent
- 75–89: stabil
- 60–74: brauchbar
- 40–59: riskant
- 0–39: kritisch

## Regel

Der Health Score ist ein Werkzeug zur Priorisierung, keine absolute Wahrheit.
