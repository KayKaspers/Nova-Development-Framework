# Release Helper Concept

## Zweck

Der Release Helper unterstützt bei der Vorbereitung eines Releases.

## Schritte

1. Version bestimmen.
2. Changelog prüfen.
3. Release Notes erstellen.
4. Quality Gates prüfen.
5. Project Brain aktualisieren.
6. GitHub Release vorbereiten.
7. Lessons Learned dokumentieren.

## Release-Arten

- Foundation Release
- Documentation Release
- Toolkit Release
- Prompt Library Release
- Template Library Release
- Project Adapter Release

## Ergebnis

Ein Release ist erst fertig, wenn nicht nur Code oder Dokumente vorhanden sind, sondern auch der Projektstand nachvollziehbar dokumentiert ist.
