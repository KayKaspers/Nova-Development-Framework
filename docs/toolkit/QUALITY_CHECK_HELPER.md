# Quality Check Helper

## Ziel

Der Quality Check Helper beschreibt, welche Prüfungen vor Commit, Release oder Projektwechsel notwendig sind.

## Vor Commit

- [ ] Nur gewünschte Dateien geändert
- [ ] Keine Secrets
- [ ] Dokumentation geprüft
- [ ] Prompt-Rückmeldung vorhanden
- [ ] Commit-Nachricht verständlich

## Vor Release

- [ ] README aktuell
- [ ] Changelog aktuell
- [ ] Release Notes erstellt
- [ ] Quality Gates erfüllt
- [ ] Project Brain aktualisiert
- [ ] Offene Risiken dokumentiert

## Vor Projektübergabe an Claude

- [ ] Ziel klar
- [ ] Kontext vollständig
- [ ] Grenzen definiert
- [ ] Rückmeldung an Nova gefordert
- [ ] Arbeitspaket klein genug
