# Prompt Selector Concept

## Zweck

Der Prompt Selector hilft, den passenden Prompt für ein Arbeitspaket auszuwählen.

## Auswahlkriterien

- Projekttyp
- Arbeitstyp
- Risiko
- gewünschte Rolle
- betroffene Module
- benötigte Rückmeldung
- NDF-Level

## Arbeitstypen

- Project Review
- Architecture Review
- Small Fix
- Feature Implementation
- Documentation Task
- Test Review
- Docker Review
- Security Review
- Release Review
- Project System Creation

## Beispiel

Wenn das Ziel ist:

> Bestehendes Projekt analysieren

Dann wähle:

```text
framework/prompts/core/PROJECT_REVIEW.md
```

Wenn das Ziel ist:

> Kleine Fehlerbehebung

Dann wähle:

```text
framework/prompts/development/SMALL_FIX.md
```
