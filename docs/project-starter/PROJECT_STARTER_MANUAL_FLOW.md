# NDF Project Starter Manual Flow v0.1

## Zweck

Der Manual Flow beschreibt, wie ein Projekt ohne Automatisierung nach NDF gestartet oder angebunden wird.

Er ist die Grundlage für den späteren NDF Generator.

## Wann nutzt man diesen Flow?

- bei einem neuen Projekt
- bei einem bestehenden Projekt
- bei einem Fork
- bei einer Dokumentationsplattform
- bei einer Suite wie CastCore, SpeakCore, AirCore oder SC-OrgaSuite

## Ergebnis

Nach dem Flow existieren:

- Project Manifest
- Project Profile
- Capability Matrix
- Project Brain
- Work Package Queue
- erste Quality Gates
- passende Prompt-Auswahl
- erste Roadmap
- initialer Health Score

## Ablauf

```text
1. Projektidee klären
2. Projekttyp bestimmen
3. NDF-Level wählen
4. Project Manifest erstellen
5. Project Profile erstellen
6. Capability Matrix ausfüllen
7. Project Brain starten
8. Quality Gates auswählen
9. Prompt-Set wählen
10. Work Package Queue erstellen
11. Health Score schätzen
12. erster Commit
```

## Grundsatz

Ein Projekt darf klein starten. Es muss nicht sofort perfekt sein. Wichtig ist, dass es nachvollziehbar, prüfbar und erweiterbar startet.
