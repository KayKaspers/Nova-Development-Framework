# NDF Project Starter Questionnaire

## 1. Projektidentität

1. Wie heißt das Projekt?
2. Wie lautet der kurze Slug?
3. Ist es neu oder bestehend?
4. Wer ist Project Owner?
5. Wer übernimmt Architektur und Review?
6. Soll Claude bei der Umsetzung helfen?

## 2. Projektziel

1. Welches Problem löst das Projekt?
2. Für wen ist das Projekt gedacht?
3. Was ist der wichtigste Nutzen?
4. Was soll das Projekt ausdrücklich nicht tun?

## 3. Projekttyp

- [ ] Web-App
- [ ] API
- [ ] Dashboard
- [ ] Docker-Suite
- [ ] Dokumentationsprojekt
- [ ] CLI/Tool
- [ ] Raspberry-Pi/IoT-Projekt
- [ ] Medien-/Streaming-Projekt
- [ ] anderes

## 4. Technik

1. Frontend?
2. Backend?
3. Datenbank?
4. Docker erforderlich?
5. CI/CD erforderlich?
6. Self-Hosting?
7. Mehrsprachigkeit?

## 5. Qualität

1. Sind Tests erforderlich?
2. Ist Security wichtig?
3. Gibt es sensible Daten?
4. Soll es Releases geben?
5. Soll das Projekt öffentlich werden?

## 6. NDF-Level

- [ ] Level 1 – Basisdokumentation
- [ ] Level 2 – Standards + Project Brain
- [ ] Level 3 – Quality Gates
- [ ] Level 4 – Health Score
- [ ] Level 5 – vollständige NDF-Konformität

## 7. Erste Arbeit

1. Was ist der erste sinnvolle Schritt?
2. Was ist das größte Risiko?
3. Was soll Claude zuerst tun?
4. Was soll Nova zuerst prüfen?
