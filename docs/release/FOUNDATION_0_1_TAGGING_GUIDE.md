# Foundation 0.1 Tagging Guide

## Tag Name

```text
v0.1.0-foundation
```

## Recommended GitHub Desktop Flow

1. Ensure all changes are committed.
2. Push origin.
3. Open the repository on GitHub.
4. Check that the latest commit is visible.
5. Create a new release from GitHub UI.
6. Use tag `v0.1.0-foundation`.
7. Use the GitHub Release Draft content.
8. Publish only after final review.

## Command Line Alternative

Only if comfortable with Git:

```bash
git tag v0.1.0-foundation
git push origin v0.1.0-foundation
```

## Warning

Do not create the tag until the release package has been reviewed.
