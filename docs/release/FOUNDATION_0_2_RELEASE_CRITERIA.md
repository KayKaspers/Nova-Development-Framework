# Foundation 0.2 Release Criteria

## Pflichtkriterien

Foundation 0.2 kann veröffentlicht werden, wenn:

- [ ] Repository Consolidation Review durchgeführt wurde.
- [ ] Basic Automation Plan existiert.
- [ ] erste Check-Strategie für Markdown/Links/Struktur existiert.
- [ ] Reference Project Integration Plan existiert.
- [ ] Reference Project System Baseline existiert oder vorbereitet ist.
- [ ] Academy Band 1 Kapitel 1–3 ausgearbeitet sind.
- [ ] MkDocs Prototype Plan existiert.
- [ ] Export Prototype Plan existiert.
- [ ] Project Brain für Foundation 0.2 aktualisiert wurde.
- [ ] Foundation 0.2 Release Notes vorbereitet sind.

## Optionale Kriterien

- [ ] erster GitHub Actions Draft
- [ ] lokaler MkDocs Build getestet
- [ ] erster DOCX-Test erzeugt
- [ ] erster Reference Project Health Score erstellt
- [ ] Lessons Learned zurück ins NDF übernommen

## Nicht erforderlich

- vollständige Website
- vollständige Academy
- vollständige CLI
- vollständige Projektintegration aller Projekte

## Release-Typ

```text
Foundation Stabilization Release
```
