# Website Content Rules

## Grundsatz

Die Website ist nicht einfach ein Spiegel des Repositories. Sie ist die nutzerfreundliche Darstellung des NDF.

## Regeln

### Jede Seite braucht einen Zweck

Eine Seite muss klar beantworten:

- Worum geht es?
- Für wen ist es?
- Was soll der Leser danach tun?

### Einsteiger zuerst

Wenn ein Thema komplex ist, beginnt die Seite mit einer einfachen Erklärung.

### Keine Sackgassen

Jede Seite sollte am Ende einen nächsten Schritt anbieten.

### Kurze Abschnitte

Lange Textwände vermeiden.

### Beispiele einbauen

Wann immer möglich:

- Beispiel
- Checkliste
- typischer Fehler
- nächster Schritt

### Repository bleibt Quelle

Die Website rendert Inhalte aus Markdown. Inhalte werden nicht direkt im Website-Build gepflegt.
