# NDF Website Concept v0.1

## Zweck

Die NDF Website soll die öffentliche und interne Dokumentationsoberfläche des Nova Development Frameworks werden.

GitHub bleibt das Repository. Die Website wird die lesbare, strukturierte und suchbare Darstellung.

## Zielgruppen

### Einsteiger

Menschen, die Softwareentwicklung, GitHub, VS Code und KI-Unterstützung lernen möchten.

### Projektleiter

Menschen, die Projekte mit NDF strukturieren, planen und prüfen möchten.

### Entwickler

Menschen, die Standards, Prompts, Templates und Workflows praktisch nutzen möchten.

### Maintainer

Menschen, die NDF weiterentwickeln oder eigene Projekte anbinden möchten.

## Hauptziele

- schneller Einstieg
- klare Navigation
- gute Suche
- strukturierte Academy
- klare Referenzbereiche
- professionelle Darstellung
- spätere Versionierung
- einfache Veröffentlichung über GitHub Pages

## Empfohlene technische Basis

Kurzfristig:

- Markdown im Repository
- GitHub als Hauptquelle

Mittelfristig:

- MkDocs Material
- GitHub Pages

Langfristig optional:

- Docusaurus oder eigene Plattform
- versionierte Dokumentation
- interaktive Diagramme
- NDF Health Dashboard
