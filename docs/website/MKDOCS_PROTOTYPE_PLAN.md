# MkDocs Prototype Plan

## Zweck

Foundation 0.2 soll die NDF-Website praktisch vorbereiten.

## Ziel

Ein lokaler MkDocs-Prototyp soll zeigen, dass NDF aus Markdown als Website darstellbar ist.

## Empfohlene Toolchain

```text
Python
MkDocs
MkDocs Material
Markdown
GitHub Pages später optional
```

## Prototype-Ziele

- lokale Website bauen
- Navigation testen
- README/Docs/Academy einbinden
- Suchfunktion prüfen
- Branding grob anwenden
- keine vollständige Website erzwingen

## Mindestnavigation

```text
Home
Getting Started
Foundation
Academy
Toolkit
Prompt Library
Template Library
Project System
Examples
Reference
```

## Nicht-Ziele

- keine finale Domain
- kein perfektes Theme
- keine vollständige Versionierung
- keine automatische Veröffentlichung als erster Schritt

## Risiken

- MkDocs braucht saubere relative Links.
- Repository-Struktur muss zur Website-Navigation passen.
- Zu viele Seiten auf einmal können unübersichtlich werden.

## Foundation 0.2 Mindestziel

Ein dokumentierter lokaler Prototyp oder ein klarer MkDocs-Buildplan.
