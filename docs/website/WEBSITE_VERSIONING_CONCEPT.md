# Website Versioning Concept

## Warum Versionierung?

Das NDF wird sich weiterentwickeln. Nutzer müssen wissen, auf welche Version sich eine Anleitung bezieht.

## Kurzfristig

Keine komplexe Versionierung. Aktueller Stand ist `main`.

## Mittelfristig

Versionen:

- latest
- foundation-0.1
- foundation-0.2
- 1.0

## Langfristig

Versionierte Dokumentation über:

- MkDocs Material Versioning Plugin
- separate Release-Branches
- Git Tags
- statische Builds pro Version

## Regel

Stabile Releases brauchen passende Dokumentation.
