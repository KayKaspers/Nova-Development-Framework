# Repository Excellence

Ein NDF-Repository ist nicht nur ein Speicherort für Dateien. Es ist die zentrale Projektzentrale.

## Ziele
- nachvollziehbare Entscheidungen
- saubere Dokumentation
- wiederholbarer Projektstart
- einfache Mitarbeit
- klare Qualitätssicherung
- menschlich kontrollierte Veröffentlichung

## Grundregeln
1. Jede wichtige Entscheidung wird als ADR dokumentiert.
2. Jeder Sprint aktualisiert den Project Brain.
3. Jede Datei hat einen klaren Zweck.
4. Änderungen werden vor dem Commit geprüft.
5. GitHub Desktop ist der empfohlene Einsteigerweg.
6. Cursor ist optional, nicht verpflichtend.
7. KI darf Inhalte erzeugen, aber keine unkontrollierten Repository-Operationen durchführen.

## Empfohlene Repository-Zonen
- `docs/` – fachliche und technische Dokumentation
- `academy/` – Lerninhalte
- `standards/` – verbindliche Standards
- `templates/` – wiederverwendbare Vorlagen
- `project-brain/` – Projektwissen
- `branding/` – visuelle Identität
- `.github/` – GitHub-Vorlagen und Workflows
- `scripts/` – Hilfsskripte
