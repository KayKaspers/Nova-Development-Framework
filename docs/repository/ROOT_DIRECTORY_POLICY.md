# Root Directory Policy

## Zweck

Der Root des Repositories soll übersichtlich bleiben.

## Erlaubte Root-Dateien

- README.md
- CONTRIBUTING.md
- CODE_OF_CONDUCT.md
- SECURITY.md
- LICENSE.md
- CHANGELOG.md
- ROADMAP.md
- .gitignore
- .gitattributes

## Nur temporär erlaubt

- Import-Anleitungen
- Sprint-READMEs
- Paket-Changelogs
- temporäre Notizen

Diese sollen nach Import entweder entfernt oder in passende Unterordner verschoben werden.

## Nicht erlaubt

- unbenannte Notizen
- doppelte README-Dateien
- generierte Exporte ohne Zielordner
- Screenshots im Root
- ZIP-Dateien im Repository

## Regel

Wenn eine Datei nicht innerhalb von 10 Sekunden erklärbar ist, gehört sie nicht in den Root.
