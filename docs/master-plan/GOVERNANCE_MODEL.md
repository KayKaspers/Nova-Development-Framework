# NDF Governance Model

## 1. Grundsatz

Das NDF wird menschlich geführt und KI-unterstützt entwickelt.

## 2. Entscheidungsrechte

Der Maintainer ist finaler Entscheider für:

- Veröffentlichung
- Projektprioritäten
- Repository-Änderungen
- externe Sichtbarkeit
- Lizenzentscheidungen

Nova empfiehlt und entwirft:

- Architektur
- Standards
- Roadmap
- Qualitätsregeln
- Prompts
- Reviews

Claude unterstützt bei:

- Umsetzung
- Textproduktion
- Code
- Automatisierung
- Detailarbeit

## 3. Repository-Regel

KI darf keine unverifizierten Repository-Operationen durchführen.

Verboten ohne menschliche Kontrolle:

- Push
- Force Push
- Reset
- Clean
- Rebase
- Branch löschen
- Tag löschen

## 4. Änderungsprozess

Jede größere Änderung benötigt:

1. Ziel
2. Begründung
3. betroffene Dateien
4. Qualitätsprüfung
5. Commit durch den Maintainer

## 5. ADR-Pflicht

Wichtige Entscheidungen werden als ADR dokumentiert.
