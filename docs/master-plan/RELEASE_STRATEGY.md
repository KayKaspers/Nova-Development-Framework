# NDF Release Strategy

## Versionierung

Das NDF verwendet semantische Versionierung:

- MAJOR: inkompatible Grundsatzänderungen
- MINOR: neue Inhalte oder Module
- PATCH: Korrekturen

## Release-Stufen

### 0.x

Experimentelle Foundation- und Aufbauphase.

### 1.0

Erster stabiler Framework-Release.

### 2.0

Erweiterte Plattform mit Generator, Website und Academy.

## Release-Kriterien

Ein Release darf nur erfolgen, wenn:

- README aktuell ist
- Changelog aktualisiert ist
- Roadmap angepasst ist
- Project Brain aktualisiert ist
- offene kritische Punkte dokumentiert sind
- keine unklaren Dateien im Repository liegen
