# NDF Quality Gates

## Gate 1 – Struktur

- klare Ordnerstruktur
- keine losen Dateien ohne Zweck
- README vorhanden

## Gate 2 – Dokumentation

- jede größere Änderung ist dokumentiert
- Project Brain aktualisiert
- Roadmap aktualisiert, falls relevant

## Gate 3 – Nachvollziehbarkeit

- Commit-Nachricht folgt Conventional Commits
- wichtige Entscheidungen haben ADRs

## Gate 4 – Sicherheit

- keine Secrets
- keine API-Schlüssel
- keine privaten Zugangsdaten

## Gate 5 – Einsteigerfreundlichkeit

- verständliche Sprache
- klare Schritt-für-Schritt-Anleitungen
- Fachbegriffe werden erklärt

## Gate 6 – Wartbarkeit

- modularer Aufbau
- keine unnötige Komplexität
- spätere Erweiterbarkeit berücksichtigt
