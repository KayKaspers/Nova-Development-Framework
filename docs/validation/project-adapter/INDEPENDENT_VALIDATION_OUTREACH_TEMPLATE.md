## DE – Anfrage

> Betreff: Kurze unabhängige Review-Anfrage — öffentliches Doku-Framework (60–120 Min.)

Hallo `<independent-reviewer>`,

ich pflege ein öffentliches, dokumentationsgetriebenes Entwicklungs-Framework (Nova Development Framework, NDF). Ein Kernstück — der „Project Adapter", eine Anleitung zur strukturierten Projektaufnahme — wurde bisher nur von den Beteiligten selbst validiert. Ich suche eine unabhängige Sicht.

**Worum es geht:** Du prüfst anhand der öffentlichen Dokumente und eines neutralen Beispielprojekts, ob die Anleitung von außen verständlich und anwendbar ist — ein Lese-/Trockenlauf, keine Code-Arbeit.

**Aufwand:** 60–120 Minuten, einmalig, Zeitpunkt frei wählbar bis `<response-deadline>`.

**Was du nicht brauchst/musst:** keine Schreibrechte, keine Installation, kein Vorwissen über das Framework, keine Code-Ausführung, keine personenbezogenen Angaben — dein Feedback wird nur mit neutraler Rollenbezeichnung (z. B. „external reviewer") verwendet.

**Wichtig:** Alles ist öffentlich und neutral — es gibt keine privaten Daten, und bitte auch keine echten Namen/Organisationen/Domains ins Feedback schreiben.

Wäre das etwas für dich? Rückmeldung gern bis `<response-deadline>`.

Danke und viele Grüße
`<maintainer>` · `<date>`

## EN – Request

> Subject: Short independent review request — public documentation framework (60–120 min)

Hello `<independent-reviewer>`,

I maintain a public, documentation-driven development framework (Nova Development Framework, NDF). A core piece — the "project adapter", a guide for structured project onboarding — has so far only been validated by the people who built it. I am looking for an independent perspective.

**What it is:** Using the public documents and a neutral example project, you check whether the guide is understandable and applicable from the outside — a reading/dry run, no coding.

**Effort:** 60–120 minutes, one-time, at your convenience until `<response-deadline>`.

**What you get:** the repository link (`<repository-link>`), a short brief (`docs/validation/project-adapter/INDEPENDENT_VALIDATOR_BRIEF.md`), a step-by-step runbook, and a feedback form.

**What you do not need:** no write access, no installation, no prior framework knowledge, no code execution, no personal data — your feedback is used with a neutral role label only (e.g. "external reviewer").

**Important:** Everything is public and neutral — there is no private data, and please do not put real names/organizations/domains into the feedback either.

Would this be something for you? A reply by `<response-deadline>` would be great.

Thanks and best regards
`<maintainer>` · `<date>`
