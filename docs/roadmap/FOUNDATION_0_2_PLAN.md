# NDF Foundation 0.2 Plan

## Status

Planned

## Prepared

2026-06-29

## Ausgangspunkt

Foundation 0.1 wurde als erster stabiler Foundation-Meilenstein veröffentlicht.

Foundation 0.2 baut darauf auf und verschiebt den Fokus:

```text
von: Framework aufbauen
zu: Framework beweisen
```

## Ziel von Foundation 0.2

Foundation 0.2 soll zeigen, dass NDF praktisch nutzbar, überprüfbar und auf echte Projekte anwendbar ist.

## Hauptziele

1. Repository weiter konsolidieren.
2. Basic Automation Checks vorbereiten oder einführen.
3. CastCore als erste echte NDF-Integration anbinden.
4. Academy Band 1 mit echten Kapiteln beginnen.
5. MkDocs Website-Prototyp vorbereiten.
6. PDF/DOCX Export-Prototyp vorbereiten.
7. Foundation 0.2 Release-Kriterien definieren.

## Nicht-Ziele

Foundation 0.2 soll nicht:

- eine vollständige CLI liefern
- die Academy komplett fertigstellen
- eine perfekte Website veröffentlichen
- alle Projekte gleichzeitig integrieren
- Foundation 0.1 nachträglich umbauen
- neue große Konzepte ohne Validierung hinzufügen

## Arbeitsprinzip

```text
klein → prüfbar → dokumentiert → wiederverwendbar
```

## Empfohlene Reihenfolge

1. Repository Consolidation
2. Basic Automation Checks
3. CastCore Integration
4. Academy Band 1 Kapitel 1–3
5. MkDocs Prototype
6. Export Prototype
7. Foundation 0.2 Release Review

## Foundation 0.2 Erfolgskriterien

Foundation 0.2 ist erfolgreich, wenn:

- das Repository sichtbar sauberer ist
- Basisprüfungen für Markdown/Links/Struktur geplant oder aktiv sind
- CastCore als echtes Projekt ein NDF Project System besitzt
- Academy Band 1 mindestens die ersten 3 Kapitel ausführlich enthält
- ein erster Website-Prototyp lokal nachvollziehbar ist
- ein erster Export-Prototyp dokumentiert ist
- Lessons Learned zurück ins NDF fließen
