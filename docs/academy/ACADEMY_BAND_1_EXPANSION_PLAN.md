# Academy Band 1 Expansion Plan

## Zweck

Foundation 0.2 soll aus dem Academy-Skeleton erste echte Lerninhalte machen.

## Ziel

Kapitel 1–3 vollständig ausarbeiten:

1. Willkommen in der Softwareentwicklung
2. Was ist das Nova Development Framework?
3. Die Rollen im NDF

## Qualitätsziel

Die Kapitel sollen für absolute Einsteiger verständlich sein.

## Kapitelstruktur

Jedes Kapitel enthält:

- Lernziel
- einfache Erklärung
- Warum ist das wichtig?
- Schritt-für-Schritt
- Beispiel
- typische Fehler
- Checkliste
- Übung
- Lösung
- Zusammenfassung

## Kapitel 1 – Willkommen in der Softwareentwicklung

Ziel:

Softwareentwicklung als strukturierten Prozess erklären, nicht nur als Programmieren.

Kernaussagen:

- Software entsteht in Schritten.
- Dokumentation ist Teil der Arbeit.
- Fehler sind normal.
- Versionierung schützt Arbeit.
- KI kann helfen, braucht aber klare Führung.

## Kapitel 2 – Was ist NDF?

Ziel:

NDF einfach erklären.

Kernaussagen:

- NDF ist ein Arbeitsrahmen.
- NDF ordnet Projekte.
- NDF hilft bei KI-gestützter Entwicklung.
- NDF macht Arbeit prüfbar.

## Kapitel 3 – Rollen im NDF

Ziel:

Maintainer, Nova, Claude, VS Code, GitHub Desktop und GitHub verständlich erklären.

Kernaussagen:

- Der Maintainer entscheidet.
- Nova plant und reviewed.
- Claude setzt kleine Arbeitspakete um.
- VS Code zeigt Dateien.
- GitHub Desktop verwaltet Änderungen.
- GitHub speichert den Stand.

## Foundation 0.2 Mindestziel

Kapitel 1–3 inhaltlich ausarbeiten und reviewfähig machen.
