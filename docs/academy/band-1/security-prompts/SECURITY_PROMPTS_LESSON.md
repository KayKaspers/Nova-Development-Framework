# Academy Lesson – Security Prompts

## Ziel

Verstehen, warum Security-Prompts anders aufgebaut sein müssen als normale Coding-Prompts.

## Grundsatz

```text
Security-Arbeit braucht Scope, Tests und klare Grenzen.
```

## CastCore-Beispiel

1. Security Baseline Review fand SEC-01.
2. Focused Security Code Fix mitigierte SEC-01.
3. Tests liefen grün.
4. Danach wurde Health Score aktualisiert.

## SpeakCore-Beispiel

Backup Delete wurde nicht direkt gebaut.

Erst Blueprint, dann Review, dann Implementierung.

## Wichtig

Ein Security-Review verbessert Sichtbarkeit.

Ein Security-Fix verbessert Schutz.

Ein Health-Score-Update dokumentiert den neuen Zustand.

## Merksatz

```text
Review ist nicht Fix. Fix ist nicht Score. Score ist nicht Release.
```
