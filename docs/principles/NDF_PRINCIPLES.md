# NDF Principles v1

## 1. Documentation First
Dokumentation ist kein Nachtrag, sondern Teil der Entwicklung.

## 2. Human Verified
Menschen treffen finale Entscheidungen, besonders bei Git, Releases und Sicherheit.

## 3. AI Assisted
KI unterstützt Planung, Umsetzung und Review, ersetzt aber nicht Verantwortung.

## 4. Modular by Default
Alles wird so gebaut, dass Bausteine wiederverwendbar sind.

## 5. Security by Design
Sicherheit wird von Anfang an mitgedacht.

## 6. Open Knowledge
Wissen wird dokumentiert, damit es nicht in Chats verloren geht.

## 7. Automation with Purpose
Automatisierung wird eingesetzt, wenn sie Qualität oder Wiederholbarkeit verbessert.

## 8. Learn Continuously
Jedes Projekt verbessert das Framework.

## 9. Quality over Speed
Stabilität, Wartbarkeit und Klarheit sind wichtiger als schnelle Ergebnisse.

## 10. Build for the Future
Entscheidungen sollen auch in mehreren Jahren noch nachvollziehbar sein.
