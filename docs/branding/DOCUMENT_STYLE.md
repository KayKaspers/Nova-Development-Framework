# Document Style

## Ziel

NDF-Dokumente sollen professionell, ruhig und gut lesbar sein.

## Aufbau

Empfohlener Aufbau:

1. Titel
2. Zweck
3. Zielgruppe oder Kontext
4. Hauptinhalt
5. Beispiel
6. Checkliste oder Kriterien
7. Nächster Schritt

## Markdown-Stil

- eine H1 pro Datei
- klare H2-Abschnitte
- kurze Absätze
- Tabellen nur für echte Struktur
- Codeblöcke mit Sprache
- Checklisten für prüfbare Schritte

## PDF/Word-Stil

- klares Titelblatt
- Inhaltsverzeichnis
- Kapitelnummerierung optional
- Kopf-/Fußzeile mit Version
- einheitliche Infoboxen

## Infoboxen

```markdown
> [!NOTE]
> Hinweis

> [!WARNING]
> Warnung

> [!TIP]
> Tipp
```

## Regel

Ein Dokument soll auch ohne Vorwissen verständlich bleiben.
