# Support Development Style

## Zweck

NDF und die daraus entstehenden Projekte können optional einen zentralen Support-Hinweis enthalten.

## Grundsatz

Support ist freiwillig, professionell und nicht aufdringlich.

## Empfohlener Text

```markdown
## Support Development

If this project helps you, you can optionally support the ongoing development of the project ecosystem.

Support is voluntary and does not unlock paid features, priority access or hidden functionality.
```

## Platzierung

Geeignet:

- README unten
- Website Footer
- About-Seite
- Dokumentation
- GitHub Repository Metadata

Nicht geeignet:

- Popups
- Paywalls
- aggressive Banner
- blockierende Hinweise

## Regel

Der Support-Hinweis darf nie den Nutzwert oder die Offenheit des Projekts einschränken.
