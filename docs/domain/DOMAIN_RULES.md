# NDF Domain Rules

## Rule 1 – Kein Objekt ohne Zweck

Jedes Domain-Objekt braucht einen klaren Zweck.

## Rule 2 – Keine Regel ohne Prüfbarkeit

Regeln müssen überprüfbar sein.

## Rule 3 – Keine Entscheidung ohne Kontext

Entscheidungen brauchen Begründung und Konsequenzen.

## Rule 4 – Kein Work Package ohne Grenzen

Jedes Work Package muss klare Grenzen enthalten.

## Rule 5 – Kein Prompt ohne Rückmeldung

NDF-Prompts fordern immer eine Rückmeldung an Nova.

## Rule 6 – Kein Projekt ohne Project Brain

Ein nach NDF geführtes Projekt braucht ein Langzeitgedächtnis.

## Rule 7 – Keine Standards ohne Lifecycle

Standards müssen verbessert, ersetzt oder archiviert werden können.
