# SampleProject – Roadmap (Ist-Zustand: unklar)

> Fiktive Roadmap. Bildet bewusst unklare Prioritäten ab.

## Aktueller Stand

Es gibt keine schriftliche Roadmap. Der Maintainer hat eine Wunschliste im Kopf; das Team wirft in Chats Ideen ein. Nichts ist priorisiert, nichts hat Kriterien.

## Nächste gewünschte Schritte (unsortierte Wunschliste)

- Notiz-Löschung „endlich sicher machen"
- individuelle Accounts statt geteiltem Admin
- README reparieren, damit Neue selbst installieren können
- CI wieder grün bekommen
- „vielleicht Tags/Labels für Aufgaben"
- „irgendwann dunkles Theme"

## Unklare Prioritäten

Sicherheit vs. Features vs. Doku — bisher gewinnt immer das zuletzt genannte Feature. Es gibt keinen Mechanismus, der Reihenfolge und Größe der Arbeit begrenzt.

## Warum NDF helfen soll

- Standortbestimmung mit Evidenz (Health Score) statt Bauchgefühl.
- Kleine, typisierte Work Packages statt „mach mal alles besser".
- Destructive-Action-Prozess für die Löschfunktion, bevor daran gearbeitet wird.
- Eine Roadmap, die aus einer Work Package Queue entsteht — nicht aus Chat-Ideen.
