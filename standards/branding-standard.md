# NDF Branding Standard

## Purpose

This standard defines how NDF and NDF-based projects should present themselves visually and structurally.

## Mandatory Rules

1. Every project must have a clear mission statement.
2. Every project must have a README header section.
3. Every project must define colors, typography and logo usage.
4. Every diagram must use a consistent style.
5. Branding must never reduce accessibility.

## README Header Pattern

```md
# Project Name

> Short project slogan.

Brief explanation in one paragraph.

## Status

Current release status and stability information.
```

## Accessibility

- Use sufficient contrast.
- Do not communicate status by color alone.
- Use alt text for images.
- Keep banners readable on mobile.
