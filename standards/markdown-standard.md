# Markdown Standard

## Regeln
- Eine H1 pro Datei.
- Kurze Absätze.
- Aussagekräftige Überschriften.
- Listen nur, wenn sie Übersicht schaffen.
- Codeblöcke mit Sprache markieren.
- Fachbegriffe erklären.

## Dateinamen
- Kleinbuchstaben bevorzugt.
- Bindestriche statt Leerzeichen.
- Für zentrale Dokumente sind Großbuchstaben erlaubt: README.md, CHANGELOG.md.
