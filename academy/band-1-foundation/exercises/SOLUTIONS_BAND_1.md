# Lösungen – Band 1

## Lösung 1

Ein Repository ist ein Projektordner mit Versionshistorie. Es speichert Dateien und Änderungen. Über GitHub kann das Repository online gesichert und geteilt werden.

## Lösung 2

Ein Commit speichert einen nachvollziehbaren Stand. Dadurch kann man später sehen, was geändert wurde. Gute Commits helfen bei Fehlersuche und Zusammenarbeit.

## Lösung 3

Vor einem Commit prüfen:

- Sind nur gewünschte Dateien geändert?
- Sind keine Secrets enthalten?
- Ist die Commit-Nachricht verständlich?

## Lösung 4

Beispiel:

> Ergänze die README um eine kurze Installationsanleitung. Keine Codeänderungen. Am Ende Rückmeldung an Nova.

## Lösung 5

Beispiel:

> README wurde erweitert. Geändert wurde nur README.md. Keine Tests nötig, da reine Dokumentation. Nächster Schritt: Nova Review.
