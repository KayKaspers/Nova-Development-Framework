# ADR-0011: Repository Release Readiness

## Status

Accepted

## Context

The NDF repository has grown from initial setup into a structured framework. To become maintainable and presentable, it needs standard repository governance files and a release-readiness baseline.

## Decision

Introduce repository-level readiness assets:

- professional README
- CONTRIBUTING
- CODE_OF_CONDUCT
- SECURITY
- GitHub issue templates
- pull request template
- .gitattributes
- Foundation 0.1 definition
- Foundation 0.1 draft release notes

## Consequences

- repository becomes easier to understand
- contribution workflow becomes clearer
- release milestone becomes measurable
- line ending behavior becomes more stable across Windows and Linux
