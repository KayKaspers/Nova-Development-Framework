# NDF Context Pack Template

> Wiederverwendbare Vorlage für einen kompakten Phasen-Snapshot (NDF-WP-113, Context Economy WP-109). Ein Context Pack beschleunigt den Einstieg in ein Folge-WP — er **ersetzt keine** Prüfung der aktuellen Dateien, keinen Public Quality Gate und kein Human-Maintainer-Review. Nur geprüfte, öffentliche Repo-Inhalte. / Reusable compact phase snapshot; never a replacement for checking current files, the public quality gate, or human-maintainer review.

## Status

`scope-locked | released | planning` · nicht v1.0 (sofern zutreffend) · Datum des Snapshots.

## Scope

Arbeitstitel der Phase und ein Satz zum Fokus.

## Current Phase

Welche Foundation-Phase (z. B. Foundation 0.x).

## Current Phase Status

Ein bis zwei Sätze: gelockt / released / in Arbeit; nicht released / nicht v1.0, wo zutreffend.

## Last Completed Work Package

`NDF-WP-xxx – Titel` + einzeiliges Ergebnis.

## Next Work Package

`NDF-WP-xxx – Titel`.

## Release-Blocking Work Packages

Liste mit Status (done / open).

## Optional Work Packages

Liste mit Status (optional / not activated).

## Deferred / Non-Goals

Kurze Liste der bewusst ausgeschlossenen Punkte.

## Accepted ADRs Relevant to This Phase

`ADR-00xx – Titel (Accepted)` + einzeilige Wirkung; nächste freie ADR-Nummer.

## Current Safety Boundaries

Die geltenden Sicherheitsgrenzen (z. B. fail closed, docs-only, keine autonomen Git-/Release-Aktionen, Human-Maintainer-Kontrolle).

## Known Notes / Limitations

Non-blocking Notes, v1.0-tracked Punkte, operational notes.

## Relevant Source Files

Pfade der wichtigsten Dokumente der Phase.

## Compact Context History

Die letzten 3–5 Compact Context Summaries (je 5–10 Zeilen), chronologisch — als kompakte Historie, keine Rohlogs.

## Next Prompt Recommendation

Empfohlener Prompt Mode (Full / Standard / Short, siehe [NDF Prompt Modes](../docs/agent-workflows/NDF_PROMPT_MODES.md)) für das nächste WP.

## What Must Not Be Claimed

z. B. keine Release-Behauptung vor Tagging, kein v1.0, keine aktive volle v1.x-Zusage, kein aktives Skill Pack.

## What Must Not Be Included

Keine privaten Daten; keine Secrets; keine Roh-Chatlogs; keine Chain-of-Thought; keine externen Skill-Inhalte; keine riesigen Kopien.

## Update Rules

- Context Pack regelmäßig (nach jedem abgeschlossenen WP) aktualisieren.
- Nur geprüfte, öffentliche Repo-Inhalte übernehmen.
- Compact Context Summary als Inputquelle erlaubt.
- Context Pack ersetzt nicht: Prüfung der aktuellen Dateien, Public Quality Gate, Human-Maintainer-Review.
